# RP2350 ArduPlane Build — Progress & Notes

**Date:** 2026-03-02
**Status:** Phase 1 firmware builds successfully, `.uf2` produced

---

## Build Pipeline

Hybrid **waf + cmake** architecture:

1. `waf configure --board rp2350minimal` — sets up ARM Cortex-M33 cross-compilation
2. `waf plane` — compiles all ArduPilot C++ → `libarduplane.a` + `libArduPlane_libs.a`
3. cmake links static libs with Pico SDK + FreeRTOS SMP + TinyUSB → `ardupilot.uf2`

The waf tool (`Tools/ardupilotwaf/rp2350.py`) automatically triggers the cmake step after compilation.

### Build Commands

```bash
python3 ./waf configure --board rp2350minimal
python3 ./waf plane
```

Output: `build/rp2350minimal/rp2350-build/ardupilot.uf2` (2.2MB)

### Memory Footprint

- **Text (code):** 1,126,348 bytes (~1.1MB of 4MB flash)
- **BSS (RAM):** 253,684 bytes (~248KB of 520KB SRAM)

## Issues Resolved

### FreeRTOS SMP API
`xTaskCreatePinnedToCore` (ESP-IDF) doesn't exist in RPi FreeRTOS SMP. Replaced with `xTaskCreate` + `vTaskCoreAffinitySet`.

### `struct dirent` Incomplete Type
Newlib ARM's `dirent.h` is a stub with `#error "<dirent.h> not supported"`. Defined `struct dirent` directly in `AP_Filesystem.h` for `HAL_BOARD_RP2350`, matching the ChibiOS pattern.

### `-Werror=undef`
Board header defines (`HAL_WITH_EKF_DOUBLE`, etc.) aren't guaranteed to be included before `AP_HAL_Macros.h`. Removed `-Werror=undef` from CFLAGS/CXXFLAGS (same approach as ESP32).

### TinyUSB CDC API
Direct `tud_cdc_*` calls failed because Pico SDK's TinyUSB build doesn't export CDC class functions without proper `tusb_config.h`. Replaced with Pico SDK's `stdio_usb` abstraction (`getchar_timeout_us`, `putchar_raw`, `stdio_flush`).

### VFP Register Mismatch
Waf-compiled objects use hard float (`-mfloat-abi=hard -mfpu=fpv5-sp-d16`) but cmake's `main.c` defaulted to soft float. Added matching float flags to cmake.

### Pico SDK Generated Headers
`pico/version.h` and `pico/config_autogen.h` are normally auto-generated by cmake. Created minimal stubs in the targets directory for the waf compilation pass.

## Known Issues / TODO

### ARM/Thumb Relocation Warnings
The link produces "Unknown destination type (ARM/Thumb)" and "dangerous relocation" warnings. Currently forced through via `--noinhibit-exec`. The waf objects are confirmed Thumb (`-mthumb`) and the correct multilib (`thumb/v8-m.main+fp/hard`) is selected. These warnings likely stem from missing ELF mapping symbols in the waf-built static libraries. **Needs investigation for runtime correctness.**

### Disabled Features
Many features disabled to fit the phase 1 minimal build:

- INS (accelerometer/gyro), GPS, rangefinder, beacon, proximity
- Barometer backends, compass backends (classes enabled, backends disabled)
- Camera, gimbal/mount, OSD, sprayer, gripper, landing gear, winch
- Quadplane, soaring, parachute, terrain following
- Generator, EFI, Torqeedo
- FrSky, CRSF, GHST telemetry
- MAVLink FTP, logging, scripting
